# 分组管理

分组管理允许管理员将用户分配到独立的节点组，使每个用户只能看到其所属分组内的节点。该功能实现了对不同用户群体的精细化访问控制和负载分布管理。

## 功能概述

**开启分组**后，每个用户会被自动分配到某个节点组，只能访问该节点组中的节点。**关闭分组**后，所有用户可正常访问所有节点，分组策略不生效。

页面通过以下标签页组织功能：

| 标签页 | 说明 |
|--------|------|
| **配置** | 开启/关闭分组功能，选择分组模式 |
| **节点组** | 创建和管理节点组，将节点分配到各组 |
| **平均模式** | 配置并触发用户均匀分配到各节点组 |
| **套餐模式** | 查看套餐与节点组的对应关系，并触发重新计算 |
| **流量模式** | 为各节点组设置流量阈值范围 |
| **当前分组结果** | 查看最新一次分组计算结果及用户分布情况 |
| **历史记录** | 浏览所有历史分组计算记录 |

## 配置标���页

### 开启分组

全局控制分组功能的开关（复选框）。

- **开启**：用户被分配到节点组，只能访问所属组的节点。
- **关闭**：所有用户可访问所有节点，分组策略不生效。

切换后立即生效。

### 分组模式

开启分组后，选择用户分配到节点组的方式：

#### 平均模式
将用户随机且均匀地分配到所有可用节点组，系统自动平衡各组的用户数量。

#### 套餐模式
根据用户购买的订阅套餐进行分组。每个套餐映射到一个特定的节点组，映射关系在产品设置中配置。

#### 流量模式
根据用户的累计流量使用量进行分组。每个节点组设置了流量范围（GB），用量落在该范围内的用户将被分配到对应组。

### 重置所有分组

> ⚠️ **此操作不可撤销，请谨慎操作。**

点击**重置所有分组**将执行以下操作：
- 删除所有节点组和用户组
- 将所有用户的分组 ID 重置为 0（未分组）
- 清除所有产品的节点组关联
- 清除所有节点的节点组关联

执行前会弹出确认对话框。

## 节点组标签页

管理可供用户分配的节点组。

### 表格列说明

| 列名 | 说明 |
|------|------|
| **ID** | 节点组的唯一标识符 |
| **名称** | 节��组的显示名称 |
| **描述** | 可选的描述信息 |
| **类型** | `通用`（默认）、`仅订阅`或`仅应用` |
| **参与计算** | 该组是否参与分组计算 |
| **流量范围（GB）** | 流量模式下使用的最小/最大流量阈值 |
| **排序** | 显示顺序 |

### 节点组类型

- **通用**：通用节点组，适用于所有分组模式
- **仅订阅**：仅用于套餐模式的订阅分组
- **仅应用**：用于特定应用的访问控制

### 操作

- **创建**：打开表单，添加新节点组。可配置名称、描述、类型、包含的节点、流量阈值和排序。
- **编辑**：修改已有节点组的配置。
- **删除**：删除节点组。该组内的节点和用户将被重新分配。

> 被关联到已过期套餐的节点组会显示**已过期**标签。

---

## 平均模式标签页

### 可用节点组数量

显示当前节点组的总数，该值根据实际节点组数量自动计算，不可手动修改。

### 分组重新计算

使用**平均**算法触发对所有用户的重新分配。

- **当前状态**：显示重新计算任务的状态 — 空闲 / 运行中 / 已完成 / 失败
- **进度条**：重新计算运行期间实时显示（每 2 秒自动刷新）
- **重新计算所有用户**：点击后启动重新计算任务

> ⚠️ **警告**：此操作将重新分配所有用户到新的分组，操作不可撤销。

---

## 套餐模式标签页

### 套餐-节点组对应关系

展示当前套餐与节点组的映射关系。

| 套餐名称 | → | 节点组 |
|----------|---|--------|
| 基础套餐 | → | A 组 |
| 高级套餐 | → | B 组 |

映射关系来源于各套餐产品设置中配置的节点组。若无映射数据，则显示"暂无映射数据"。

### 分组重新计算

使用**套餐**算法触发重新计算。系统根据用户当前的有效订阅套餐重新分配节点组。

> ⚠️ **警告**：此操作将重新分配所有用户到新的分组，操作不可撤销。

---

## 流量模式标签页

### 流量范围配置

可为每个节点组设置最小和最大流量使用阈值（单位：GB）。累计流量落在某组范围内的用户将被分配到该组。

可配置项：
- **最小流量（GB）**：下限（含）。留空表示无下限。
- **最大流量（GB）**：上限（含）。留空表示无上限。

各节点组的配置单独保存，互不影响。

### 分组重新计算

使用**流量**算法触发重新计算。系统根据用户的流量使用量与各组配置的阈值范围进行对比，重新分配节点组。

> ⚠️ **警告**：此操作将重新分配所有用户到新的分组，操作不可撤销。

---

## 当前分组结果标签页

展示**最新一次已完成的分组计算**的详细信息。

### 计算信息

| 字段 | 说明 |
|------|------|
| **分组模式** | 本次计算使用的模式（平均 / 套餐 / 流量） |
| **状态** | 已完成 / 运行中 / 失败 / 空闲 |
| **触发类型** | 手动触发 / 自动触发 / 定时触发 |
| **成功 / 失败数** | 成功或失败分配的用户数量 |
| **开始时间** | 计算任务的开始时间 |
| **结束时间** | 计算任务的结束时间 |

### 分组详情统计

汇总所有节点组的整体数据：
- **总用户数**：已分配用户总数
- **总节点数**：所有分组中的节点总数
- **总节点组数**：参与本次结果的节点组总数

### 各组详细数据

以表格形式展示每个节点组的详情：
- **节点组**：节点组名称及 ID
- **用户数**：点击数字可弹出对话框，查看该组内的用户列表（包含用户 ID 和邮箱）
- **节点数**：该组包含的节点数量

## 历史记录标签页

按时间倒序浏览所有历史分组计算记录。

每条记录包含：
- 分组模式、状态、触发类型
- 开始时间和结束时间
- 成功/失败用户数
- 各组详细分布情况（可通过展开行��详情视图查看）

## 使用场景示例

### 场景一：隔离高级用户

**目标**：让高级套餐用户访问质量更好的节点。

1. 前往**节点组** → 创建两个分组：`标准组` 和 `高级组`
2. 将高性能节点分配到 `高级组`
3. 在产品设置中，将高级套餐映射到 `高级组`
4. 前往**配置** → 开启分组 → 选择**套餐模式**
5. 前往**套餐模式**标签页 → 点击**重新计算所有用户**

### 场景二：均匀分担负载

**目标**：将用户负载均衡分布到多个服务器区域。

1. 按区域创建节点组（例如：`美国`、`欧洲`、`亚太`）
2. 将各区域节点分配到对应分组
3. 前往**配置** → 开启分组 → 选择**平均模式**
4. 前往**平均模式**标签页 → 点击**重新计算所有用户**

### 场景三：限流高流量用户

**目标**：将流量使用量非常高的用户迁移到单独的限速节点组。

1. 创建节点组：`普通用户`（0–100 GB）和 `重度用户`（100 GB 以上）
2. 在**流量模式**标签页中设置各组的流量范围
3. 前往**配置** → 开启分组 → 选择**流量模式**
4. 前往**流量模式**标签页 → 点击**重新计算所有用户**

---

## 注意事项

1. **修改后需手动重新计算**：更改分组模式或节点��配置后，系统不会自动重新分配用户，需手动触发重新计算。
2. **实时进度监控**：运行状态每 2 秒自动刷新一次。
3. **同时只能运行一个任务**：重新计算进行中时，无法启动新的计算任务。
4. **重置操作不可撤销**：「重置所有分组」会永久删除所有分组数据，请谨慎操作。
5. **删除节点组后会影响节点和用户**：被删除节点组内的节点和用户将被重新分配。
