# Installation

This guide covers the recommended NPanel deployment path. The current Docker workflow builds from the backend source repository and starts MySQL, Redis, HTTP, and gRPC services with Docker Compose.

## Recommended: One-Click Docker Compose

```bash
curl -fsSL https://npanel.dev/scripts/en/install-docker.sh | sudo bash && \
curl -fsSL https://npanel.dev/scripts/en/install-npanel.sh | bash
```

The installer:

- Clones [npanel-dev/NPanel-backend](https://github.com/npanel-dev/NPanel-backend)
- Builds the local `npanel:latest` image
- Writes `.env`, `docker-compose.yml`, and `configs/config.docker.yaml`
- Starts MySQL, Redis, NPanel HTTP on `8081`, and gRPC on `9012`

## Manual Docker Compose

```bash
git clone https://github.com/npanel-dev/NPanel-backend.git
cd NPanel-backend
docker compose up -d --build
```

Before production use, edit:

- `.env`
- `configs/config.docker.yaml`

## Access

- User panel: `http://your-server-ip:8081`
- Admin panel: `http://your-server-ip:8081/admin/`

Default credentials are generated by the one-click installer. If you deploy manually from the backend repo, check `configs/config.docker.yaml`.

## Operations

```bash
cd ~/npanel
docker compose ps
docker compose logs -f npanel
docker compose restart
```

## Upgrade

```bash
cd ~/npanel
git pull --ff-only
docker compose up -d --build
```
